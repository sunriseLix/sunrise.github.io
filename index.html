Hexo+Github搭建个人博客

前言

感谢大黑引领我入前端，建议我搭建博客，用于记录自己学习前端技术的过程，一来对自己所学的技术进行总结，二来也是为后来跟我一样经历的前端爱好者们做一点力所能及的贡献。技术小白多次尝试终于搭建成的博客，为初学者提供一点小小的帮助，毕竟我有点笨，搭了大半天。
–2016.08.15

正文

我是刚从windows重度用户转为os x系统的，尽管两个系统都搭建过，不过用Mac后写代码的感觉完全就不一样了，感觉自己专业了许多(哈哈，其实还是什么都不懂)，写起来更方便，体验更好了，这篇文章主要是用Mac搭建的，不过大体的步骤差不多。

软件安装

1、安装git

采用的homebrew安装git，homebrew对于OS X是不可或缺的套件管理器，在homebrew官网安装，打开终端（我用的是Iterm终端工具，用homebrew下载的)，brew install git下载安装

2、安装node

同样也是采用homebrew下载安装node，brew install -g node,当然这些都可以从官网下载安装，不过用管理器方便些。

3、安装hexo

打开Iterm，输入命令npm install hexo-cli -g，hexo官网提供了使用方法

hexo 常用的几个命令，#后面为注释。

1、hexo的配置文件

若你不喜欢hexo默认主题，可以更换其他的主题风格，创建自己的个性化博客，进入hexo官网进入theme选择自己喜欢的主题，进入该主题的Repository使用git clone克隆到本地的theme文件里，修改hexo的配置文件\_config.yml里的theme名字为自己下载的主题名即可。 在 Hexo 中有两份主要的配置文件，其名称都是 _config.yml。 其中，一份位于站点根目录下，主要包含 Hexo 本身的配置；另一份位于主题目录下，这份配置由主题作者提供，主要用于配置主题相关的选项。我更换的主题是Next，分享个网页http://theme-next.iissnan.com/getting-started.html，该文章详细的讲述了对Next主题的配置过程,可以改变博客风格，更换头像，图标等。这就是hexo搭建博客的主题个性化，多样化之处。

按照自己喜欢的样式配置好文件后，执行hexo g,hexo s本地浏览localhost:4000看效果。

写第一篇博客

整个流程就是搭建博客的大致步骤，可以参考我提供的文章辅助搭建，由于hexo写文章采用的是Markdown的写法，可以看看Markdown语法、简书中推荐的一篇文章Markdown–入门指南推荐了在什么系统下选择怎样的Markdown书写工具使用起来更好，比如我在Mac OS X系统下用的就是MOU这款工具，它很简单，小巧，界面友好，在help文档里提供了使用Markdown语法的例子，方便随时查看。掌握好基本的语法，就可以写自己的博客了，最后部署到Github上，在浏览器端就可以访问你的博客了。

总结

之前在windows系统上安装过hexo，感觉对某些主题支持的不是很好，配置时候会报错，遇到问题还是多查查资料看看，这是第一篇博客，借鉴了别人的一些描述方式，不过我都是按照这个步骤搭建成功的，写的可能不是很好，不过都是经验之谈，后期应该会添加一些功能丰富博客吧，参考我分享的那个网站，next—theme主题这个，里面就讲了怎么添加功能，当然相应的主题在Github上也能搜出相关的配置文件，掌握一个就可以随心所欲改变自己的博客了，总体来说一定要自己去尝试，继续努力吧。
